## üéØ Overview

### Search Flow (No Radius)
1. User searches ‚Üí Gets keyword suggestions
2. User clicks suggestion ‚Üí Gets ALL vendors with that keyword (full business details)
3. User submits lead with location ‚Üí Only nearby vendors (within radius) get real-time notification

### Lead Flow (With Radius + Real-time)
1. User submits lead with location + radius (default 15km)
2. System finds vendors within radius with matching keywords
3. Real-time notification sent to ALL matching vendors via Socket.IO
4. Each vendor can Accept/Reject independently
5. All responses saved in vendor's history

## üîå Socket.IO Integration (Real-time)

### Client Setup (Vendor Side)

```javascript
import io from 'socket.io-client';

// Connect with JWT token
const socket = io('http://localhost:3000', {
  auth: {
    token: 'vendor_jwt_token_here'
  }
});

// Listen for new leads
socket.on('new_lead', (data) => {
  console.log('New lead received!', data);
  
  // data structure:
  // {
  //   leadResponse: { ... },
  //   lead: {
  //     searchKeyword: "react training",
  //     description: "...",
  //     userLocation: { ... },
  //     user: {
  //       name: "Amit Kumar",
  //       phone: "+919876543210",
  //       email: "amit@example.com"
  //     }
  //   }
  // }
  
  // Show notification to vendor
  showNotification('New Lead Received!', data.lead.searchKeyword);
});

// Connection events
socket.on('connect', () => {
  console.log('Connected to server');
});

socket.on('disconnect', () => {
  console.log('Disconnected from server');
});

socket.on('error', (error) => {
  console.error('Socket error:', error);
});
```

### Client Setup (User Side)

```javascript
import io from 'socket.io-client';

const socket = io('http://localhost:3000', {
  auth: {
    token: 'user_jwt_token_here'
  }
});

// Listen for vendor responses
socket.on('lead_update', (data) => {
  console.log('Vendor responded!', data);
  
  // data structure:
  // {
  //   leadId: "lead_id",
  //   status: "accepted",
  //   message: "A vendor has accepted your lead"
  // }
  
  showNotification('Lead Update', data.message);
});
```

## üìù Key Features

‚úÖ **Search Suggestions** - No radius, just keyword suggestions
‚úÖ **Search Vendors** - With location & radius, tuta futa keyword match bhi chalega
‚úÖ **Distance Sorting** - Nearest vendors first
‚úÖ **Lead Submission** - With radius (15km default), only nearby vendors notified
‚úÖ **Real-time Notifications** - Socket.IO for instant vendor notifications
‚úÖ **Multiple Vendors** - Same lead sent to all matching vendors in radius
‚úÖ **Accept/Reject** - Each vendor responds independently
‚úÖ **History Tracking** - All responses saved
‚úÖ **Statistics** - Vendor performance metrics
‚úÖ **Flexible Search** - Spaces allowed, partial match supported
‚úÖ **Optional Description** - Not mandatory in search or lead submission

## üîÑ Complete Flow Example

### 1. User Searches
```
GET /api/lead/search-suggestions?searchTerm=react
‚Üí Gets: ["react training", "react course", ...]
```

### 2. User Views Vendors (With Location & Radius)
```
GET /api/lead/search-vendors?keyword=react train&longitude=72.8777&latitude=19.0760&radius=15000
‚Üí Gets: Nearby vendors (within 15km) with matching keywords (tuta futa match bhi chalega)
‚Üí Results sorted by distance (nearest first)
```

### 3. User Submits Lead
```
POST /api/lead/submit
Body: {
  searchKeyword: "react training",
  description: "Need weekend batches",  // OPTIONAL
  location: { longitude: 72.8777, latitude: 19.0760 },
  radius: 15000
}
‚Üí Only vendors within 15km get real-time notification
```

### 4. Vendors Receive (Real-time)
```
Socket.IO event: 'new_lead'
‚Üí All matching vendors within radius receive notification instantly
```

### 5. Vendor Responds
```
POST /api/lead/vendor/respond/response_id
Body: { status: "accepted", notes: "..." }
‚Üí Saved in vendor's history
```

### 6. User Sees Responses
```
GET /api/lead/my-leads
‚Üí Sees all vendor responses (accepted/rejected)
```
